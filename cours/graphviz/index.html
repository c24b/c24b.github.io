Quelques définitions théoriques¶

Un graphe est un ensemble de points relis par des segments.

Les points sont appels sommets ou noeuds du graphe. Nous les appelerons ici et tout au long de ce tutoriel les noeuds (nodes).

Les segments sont les artes ou les liens du graphe (edge). Nous les appelerons liens ou artes tout au long du tutoriel bien que ce ne soit pas la dnomination mathmatiques

Les somments situs chaque bout sont les extrmits de l'arte.

Nous avons en effet dj dans notre graphe la liste des noeuds, et la liste des liens. Il suffit simplement de lister tous les couples de noeuds unique possible et de dtecter les liens qui existent dj.

Le nombre de liens possibles entre les noeuds peut ici se calculer facilement, Il s'agit d'une opration d'algbre combinatoire

qu'on appelle un coefficient binominal

Cela consiste calculer dans une liste d'lement k le nombre de combinaison n.

Soit en une barbare notation mathmatique: calculer la factorielle de n sur la factorielle de n multiplie par la factorielle de n mois k

def combo_score(n,k):
    from math import factorial as fact
    #en combinatoire calculer le nombre de combinaison de 2 elements sur un ensemble de 482 elements
    #https://fr.wikipedia.org/wiki/Combinaison_(math%C3%A9matiques)
    #combin = lambda n,k: fact(n)//(fact(k)*fact(n-k))
    return fact(n)/(fact(k)*fact(n-k))

En python, les combinaisons sont gnrs grace la fonction combinations disponible dans le module itertools

Combinations produit un nombre n de combinaisons d'lement a partir d'une liste k

Par exemple: la combinaison de 2 elements pour la liste ["A", "B", "C"] nous donnera: [("A","B"), ("B","C"), ("A", "C")]

>>> from itertools import combinations
    >>> n = 2
    >>> k  = ["A", "B", "C"]
    #il s'agit d'un itrateur 
    #donc pour afficher nous devons drouler la liste
    >>> print [n for n in combinations(n, k)]

et non pas une liste de combinaison possible avec permutations telle que:

[(A,B), (B,C), (C,A), (B,A), (C,B), (A,C)]

Ordre, Degré, Voisinage¶

  • Lordre dun graphe est le nombre de sommets de ce graphe.

  • Dans un graphe, le degr de chaque sommet est le nombre de liens (artes) dont il est lune des extrmits.

      Pour caractriser le graphe on value son degr maximal et son degr minimal
  Lorsque le degr maximal et le degr minimal sont les mmes on parle de graphe rgulier
  • Quand deux noeuds sont relis par un lien (une arte) on dit qu'ils sont adjacents (ou voisins)
  • La somme des degrs des sommets est gale au double du nombre de liens (artes).

def is_regular_graph(g):
    degrees = [len(g.neighbors(n)) for n in g.nodes()]
    return bool(max(degrees) ==min(degrees))

Chaîne, cycle et parcours de graphe¶

  • Un chaine est une suite quelconque d'artes et donc de sommets adjacents.

         Une chane reproduit le parcours du graphe en partant d'un noeud sous forme de pas ou de parcours
     Un parcours de graphe est alors l'algorithme consistant  explorer les sommets d'un graphe de proche
     en proche  partir d'un sommet initial et de dfinir une chane.

  • Un cycle est une chaine dont l'origine et l'extrmit sont confondues, condition que toutes les artes soient diffrentes, il peut contenir parfois le mme sommet.

        Un cycle est un parcours entre un point A et un point B et qui emprunte un chemin unique: 
    il passe toujours par un lien (une arte) diffrent.

  • On parle de chaine ferme quand les artes ne sont pas toujours distinctes

      Une chaine ferme est donc une chaine  qui repasse par des liens dj emprunt.

Parcours de graphe¶

Pour lister tous les parcours possibles de ce graphe il fait prendre en compte que

- il y a autant de chaines que de liens (artes ou edges) d'un noeud  l'autre
- on ne doit pas repasser par un meme noeud, ni retourner sur nos pas et s'arreter  un moment prcis
pour cela il convient de dfinir:
    * la **source** ou point de dpart
    * la **cible** ou point d'arrive
    * Le nombre maximal de **pas**

Dans le cas d'un cycle donc la source et la cible sont les mmes

Une implmentation nave consisterait lister tous les noeuds: puis lister les voisins et lister ensuite les voisins des voisins et ainsi de suite jusqu' la distance maximale soit n => n1 => n2 => n3 => n4 => n5 => n... mais il faut prendre garde chaque tape que chaque noeud est bien unique et si le noeud est egal l'un des noeuds de la liste arreter le parcours et ce n'est pas si simple... Je vous propose d'essayer de l'implmenter

Sinon networkx nous propose plusieurs algorithmes de parcours de graphe Nous verrons dans le dtail les parcours de graphes un peu plus loin
